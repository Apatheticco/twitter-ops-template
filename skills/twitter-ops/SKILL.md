# Twitter 日运营 — 主调度中心

一站式运营工作流调度器。串联11个子Skill，支持全自动和人工干预两种模式。

---

## 模式控制

**一条流水线，两种跑法。**

| 模式 | 触发方式 | 行为 |
|------|---------|------|
| 🤖 自动模式 | "跑一轮" / "自动跑" / 默认 | 全流程自动执行，按预设规则决策，输出终稿 |
| 🔧 手动模式 | "手动跑一轮" / "跑一轮，手动" | 每个检查点暂停，等待用户确认/修改后继续 |
| 🔀 随时切换 | 自动模式中用户发消息 | 当前步骤变为手动，用户处理完后可选择继续自动或保持手动 |

**核心原则：默认自动，随时可拦。**

用户在任何阶段发送消息，等于触发当前检查点，流程暂停等待指令。

## 价格数据铁律（贯穿全流程）

1. **单一信源**：所有推文中出现的价格/涨跌幅，只能引用 `crypto_realtime_price_batch` 返回的数字。新闻标题、热门话题里的价格描述一律视为"历史叙述"，不作为当前数据使用。
2. **简报强制分区**：热点简报必须分为「📊 实时数据区」（只放API返回的硬数据）和「📰 叙事摘要区」（新闻/KOL观点/事件描述，标注来源）。两个区的数字不混用。
3. **写推文前价格二次确认**：Stage 3 撰写推文前，如果推文涉及具体价格，必须重新调一次 `crypto_realtime_price_batch` 获取最新数据，不引用 Stage 1 简报里的数字（采集到写稿可能已过几分钟，价格会变）。
4. **质检价格核验**：检查点③ 的质检清单新增硬性检查项：`[ ] 推文中所有价格/数据与最近一次API返回值一致`。

## 主流水线：内容生产

```
[热点采集] → 检查点① → [选题排序] → 检查点② → [价格二次确认] → [内容撰写] → 检查点③(含价格核验) → [终稿输出]
 trend-scout              topic-engine              crypto_realtime_price_batch    composer/deep-write
```

### Stage 1：热点采集
→ 调用 `trend-scout`（默认 Quick 模式）

**输出**：热点简报（CT热点 + 链上异动 + 价格快照 + 宏观信号）

#### 检查点 ①

| 模式 | 行为 |
|------|------|
| 🤖 自动 | 按热度+数据密度自动筛选 Top 3 话题，直接进入 Stage 2 |
| 🔧 手动 | 展示热点清单，等待用户选择/过滤/补充话题 |

**手动模式输出格式：**
```
━━━━━━━━━━━━━━━━━━━━━━━
⏸️ 检查点① — 热点筛选
━━━━━━━━━━━━━━━━━━━━━━━
以下热点已采集，请选择要进入选题的话题：

1. 🔴 [话题A] — 热度: XXX — 数据密度: 高
2. 🟡 [话题B] — 热度: XXX — 数据密度: 中
3. 🟡 [话题C] — 热度: XXX — 数据密度: 低
4. ⚪ [话题D] — 热度: XXX — 数据密度: 低
...

回复选项：
- 数字（如 "1 2 3"）→ 选择指定话题
- "全部" → 全部进入选题
- 补充话题（如 "加上ETH合并的话题"）
- "继续" → 用自动推荐的 Top 3
━━━━━━━━━━━━━━━━━━━━━━━
```

**自动模式决策规则：**
- 筛选 Top 3 话题
- 优先级：有链上/价格数据支撑的 > 纯新闻 > 纯观点
- 排除已在过去24h发过的同类话题（如有 content-vault 数据）

---

### Stage 2：选题排序
→ 调用 `topic-engine`（输入：筛选后的话题）

**时效性硬过滤**（topic-engine 最先执行）：>48h 直接淘汰，24-48h 降级保留需≥2个数据点。
**热点匮乏自动补充**：过滤后可用话题 < 2 个 → 自动回调 trend-scout 补充扫描（twitter_advanced_search × 2 + search_finance_news），用补充结果重新选题。

**输出**：每个话题的角度方案 + P0-P3优先级排序 + 推荐格式（单推/Thread/长文）

#### 检查点 ②

| 模式 | 行为 |
|------|------|
| 🤖 自动 | 选 P0 选题，自动匹配格式（单推/Thread/引用推），直接进入 Stage 3 |
| 🔧 手动 | 展示选题方案，等待用户挑选角度、调整优先级、指定格式 |

**手动模式输出格式：**
```
━━━━━━━━━━━━━━━━━━━━━━━
⏸️ 检查点② — 选题确认
━━━━━━━━━━━━━━━━━━━━━━━
已生成选题方案：

📌 P0（建议立即发布）
  话题：[话题A]
  角度：[锐评/数据/反向...]
  格式：单推（140字）
  理由：[时效性强 + 数据充分]

📌 P1（建议今日发布）
  话题：[话题B]
  角度：[数据驱动]
  格式：Thread（5条）
  理由：[多维度数据，适合展开]

📌 P2（可选）
  话题：[话题C]
  角度：[嘲讽/吐槽]
  格式：单推（140字）
  理由：[争议性高，有传播力]

回复选项：
- "P0" / "P1" / "全部" → 选择要写的选题
- 调整角度（如 "P0换成数据角度"）
- 调整格式（如 "P1改成单推"）
- 补充指令（如 "P0加上链上数据"）
- "继续" → 用自动推荐
━━━━━━━━━━━━━━━━━━━━━━━
```

**自动模式决策规则：**
- 选 P0 和 P1（最多2个选题同时推进）
- 格式自动匹配：
  - 时效性高（<2h） + 数据少 → 单推
  - 数据密度高 + 多维度 → Thread（3-10条）
  - 借势热推 → 引用推
- 风格自动匹配 topic-engine 的角度模板
- **注意**：流水线仅产出推文级内容。长文（deep-write）和Thread拆解（thread-builder）需用户单独触发

---

### Stage 3：内容撰写
→ 调用 `tweet-composer`（单推/Thread/引用推）

**输出**：完整草稿

#### 检查点 ③

| 模式 | 行为 |
|------|------|
| 🤖 自动 | 质检通过后直接输出终稿，等待用户发布 |
| 🔧 手动 | 展示草稿，等待用户审稿/修改/换风格 |

**手动模式输出格式：**
```
━━━━━━━━━━━━━━━━━━━━━━━
⏸️ 检查点③ — 草稿审核
━━━━━━━━━━━━━━━━━━━━━━━
[完整草稿内容]

回复选项：
- "OK" / "可以" → 确认终稿
- 修改意见（如 "开头太弱，换个hook"）
- "换个风格" / "用XXX风格重写"
- "再写一版" → 生成新版本
- "太长了" / "太短了" → 调整长度
━━━━━━━━━━━━━━━━━━━━━━━
```

**自动模式决策规则：**
- 内部运行质检清单（tweet-composer或deep-write自带）
- 质检全通过 → 直接输出终稿
- 质检有问题 → 自动修正后输出

---

### Stage 4：终稿输出

所有模式都在此暂停，等待用户确认发布。

```
━━━━━━━━━━━━━━━━━━━━━━━
✅ 终稿就绪
━━━━━━━━━━━━━━━━━━━━━━━
[最终内容]

📋 可直接复制发布
━━━━━━━━━━━━━━━━━━━━━━━
```

**发布始终需要用户确认，不论哪种模式。**

## 热点采集执行模式

trend-scout 有 6 种执行模式，主调度器根据指令自动匹配：

| 模式 | 触发 | 范围 | 耗时 |
|------|------|------|------|
| 🟢 首扫/完整 | "跑一轮" / 当日首次 | Wave 1+2 全量 | ~3-5分钟 |
| 🔵 刷新 | "刷新一下" / "有什么新的" / "更新" | 高时效源增量（8工具并行） | ~1-2分钟 |
| 🟡 快速 | "快速扫一下" | Wave 1 | ~1分钟 |
| 🔴 突发 | "突发：[事件]" | 定向搜索+价格 | ~1分钟 |
| 🟠 补充 | "热点不够" / 自动回调 | 扩源找话题 | ~1分钟 |
| 🟣 深度 | "深挖[Token/话题]" | 全部+单点深挖 | ~5-8分钟 |

### 🔵 刷新模式

首扫建立全景后，日内后续扫描只跑高时效源追增量：
→ 调用 `trend-scout`（Refresh 模式）

并行 8 路：twitter_advanced_search × 2 + search_finance_news + open_feed_news + open_feed_articles + crypto_realtime_price_batch + whale_trader_feeds + top_traders_live_24h + kol_call_orders_24h

**不跑**：Followin热榜排序/频道、TG feeds、经济日历、国债利率

**输出**：增量补丁（🆕新增 / 📈升温 / 🔄仓位变化 / 💰价格delta）→ 进入 Stage 2 选题排序

## 辅助流水线

以下流程独立于主流水线，按需或定期运行：

### 🔥 突发快速通道

```
触发："突发：[事件]" / "Breaking：[事件]"
→ trend-scout（Breaking模式，定向采集）
→ topic-engine（快速出P0角度）
→ tweet-composer（单推，15分钟内完成）
→ 检查点③ → 终稿
```

全程自动，仅在终稿暂停。争分夺秒，速度优先。

### 💬 互动管理

```
触发："互动规划" / "今天互动什么"
→ engagement-manager（生成今日Outbound目标+Inbound检查）
  → hot-comment（执行Outbound评论）
  → comment-ops（执行Inbound回复）
```

### 📊 定期复盘

```
触发："这周数据怎么样" / "周报"
→ performance-review（数据分析）
→ content-vault（高表现推文入库）
→ 输出优化建议（反馈给 topic-engine 和 tweet-composer）

触发："竞对在发什么"
→ competitor-watch（竞对分析）
```

## 人设提醒

每次运营开始前，确保所有内容符合人设（详见 `references/voice-guide.md`）：

- **强观点**，不模棱两可
- **简洁**，能一句话说完就一句话说完
- **数据为王**，用数字说话
- **反向风险必须有**，每个分析都写对立面
- **不做价格预测**，不给投资建议
- **允许适当爆粗**，有幽默感但不强行
- **速度 > 完美**，先推重要的
- **一手源 > 二手源**
