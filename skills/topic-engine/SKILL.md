# 话题引擎 Topic Engine

trend-scout 的下游、tweet-composer 的上游。将热点情报转化为可执行的推文角度。

---

## 核心定位

```
trend-scout（情报采集）→ 【topic-engine（选题+开角度）】→ tweet-composer（撰写）
```

topic-engine 不采集数据，但可以**回调 MCP 工具补充数据**来增强角度的数据支撑度。

---

## 价格数据规则

- 引用价格/涨跌幅时，只使用上游简报「📊 实时数据区」的数字（来自 `crypto_realtime_price_batch`）
- 「📰 叙事摘要区」中的价格描述视为历史叙述，不作为当前数据引用
- 如需补充实时价格，回调 `crypto_realtime_price_batch` 获取，不使用新闻标题里的数字

---

## 时效性硬性过滤（最先执行，不可跳过）

**在任何排序和开角度之前，先执行时效过滤，直接淘汰过期事件。**

| 事件时效 | 处理方式 |
|---------|---------|
| ≤ 24h | ✅ 正常进入选题流程 |
| 24h - 48h | ⚠️ 降级处理：仅当有强数据支撑（≥2个可验证数字）且仍在Twitter热议时保留，否则淘汰 |
| > 48h | ❌ 直接淘汰，不进入选题，不开角度 |

**判断时效的方法：**
- 优先使用 trend-scout 简报中标注的「时效性」字段（约X小时）
- 若无标注，根据新闻 `publish_time` 与当前时间计算
- **特例**：以下类型即便 >48h 也可保留：
  - 持续发展中的事件（每天有新进展的）→ 只写最新进展，不写陈年事实
  - 数据周期性发布（CPI、非农等）→ 只在发布当天的 24h 内有效

**过滤后若可用话题 < 2 个：**
- **自动模式**：自动回调 trend-scout 的「补充扫描」（search_finance_news + news_crypto_latest + economic_calendar），用补充结果重新进入选题流程
- **手动模式**：输出警告，等待用户选择：
```
⚠️ 时效过滤后可用话题不足（仅 X 个），可选操作：
1. 自动触发补充扫描（跨媒体新闻 + 加密快讯 + 宏观日历）
2. 重新跑完整 trend-scout 采集
3. 用户手动补充关注话题
```

## 工作流程

### 第一步：接收输入 & 话题排序

**输入来源：**
- **trend-scout 简报**（标准输入）：包含结构化热点列表、交叉信号、数据信号
- **用户直接输入**：任何话题/事件/新闻（格式自由）

**话题优先级排序（时效过滤通过后执行）：**

| 优先级 | 信号特征 | 说明 |
|--------|---------|------|
| P0 紧急 | 抢先窗口 <6h + 主流媒体未覆盖 | 必须立即开角度，时效性决定一切 |
| P1 高价值 | 多源共振（≥2源确认）+ 数据支撑强 | 优先开角度，适合数据Thread |
| P1 高价值 | KOL-大户分歧 / 数据-叙事背离 | 天然的争议性素材，推文价值最高 |
| P2 标准 | 热度高但单源 / 时效 12-24h | 正常排队处理 |
| P3 备选 | 时效 24-48h + 有数据支撑 | 降级保留，填补空缺时使用 |

**排序公式（心理模型）：**
```
话题价值 = 时效性权重(40%) × 数据支撑度(30%) × 争议性(20%) × 热度(10%)
```
时效性权重最高 — 一个有数据的6小时话题 > 一个火爆的48小时话题。

### 第二步：为每个话题选择角度模板

不再固定"锐评/数据/反向"三板斧，而是**根据话题特征匹配最优模板组合**。

**11个角度模板（详见 references/angle-templates.md）：**

| # | 模板 | 适用场景 | 时效要求 |
|---|------|---------|---------|
| 1 | 🔥 直击对立（Contrarian） | 热点共识、所有人看好/看空 | 4-24h |
| 2 | 📊 数据讲故事（Data Narrative） | 新数据发布、链上异常 | 4-48h |
| 3 | 🔄 反向风险（Hidden Risk） | 利好背后的风险信号 | 4-48h |
| 4 | 📈 历史平行线（Historical Parallel） | 市场周期、政策重演 | 24h+ |
| 5 | 😤 嘲讽吐槽（Sarcasm & Roast） | 矛盾言行、行业笑话 | 4-24h |
| 6 | 🚀 破局新视角（Novel Framework） | 转折点、新兴现象本质 | 24h+ |
| 7 | ⚡ 突发快评（Breaking React） | 重大事件刚发生 | <4h |
| 8 | 🎓 深度教育（Educational Thread） | 复杂概念、认知误区 | 24h+ |
| 9 | 🎯 对比分析（Comparative） | 竞争币种、工具对比 | 24h+ |
| 10 | 💡 交易洞察（Trading Insight） | 链上异常、市场结构变化 | 4-24h |
| 11 | 🌍 宏观视角（Macro Big Picture） | 央行政策、利率汇率 | 24h+ |

**自动匹配规则：**

```
话题类型 → 推荐模板组合

突发事件 (<6h)：
  → 模板7(快评) + 模板2(数据) + 模板3(风险)

KOL-大户分歧：
  → 模板1(对立) + 模板2(数据) + 模板10(交易洞察)

数据驱动事件（CPI/链上异常）：
  → 模板2(数据) + 模板11(宏观) + 模板3(风险)

行业争议（项目撕逼/监管冲突）：
  → 模板5(嘲讽) + 模板1(对立) + 模板6(新视角)

市场周期信号：
  → 模板4(历史平行) + 模板2(数据) + 模板10(交易洞察)

新叙事/新赛道：
  → 模板6(新视角) + 模板9(对比) + 模板8(教育)
```

每个话题生成 **3个角度**，从匹配的模板组合中各选一个。

### 第三步：生成角度 & 补充数据

对每个角度：

**3.1 核心观点**
- 一句话概括，强观点，不废话
- 必须有明确立场（不要"可能涨也可能跌"的废话）

**3.2 数据/逻辑支撑**
- 优先使用 trend-scout 已采集的数据（不重复调用）
- 数据不足时，**回调 MCP 工具补充**：

```
需要链上数据补充 →
  open_channel_feeds(quant_signal/fund_movement)
  whale_trader_feeds

需要价格/技术数据补充 →
  crypto_realtime_price_batch
  market_analyst(symbol, indicators="all")

需要KOL观点补充 →
  open_feed_list_tag_opinions(symbol)
  twitter_advanced_search(query, queryType="Top")  ← Agent子进程

需要宏观数据补充 →
  finance_tool_treasury_rates
  fred_get_series(series_id)
  search_finance_news(keyword)  ← 只取title+content
```

**3.3 反向风险（每个角度必须有）**
- 1-2句指出对立观点或潜在风险
- 这不是"两面讨好"，是展示思考深度

**3.4 开头金句**
- 推文的第一句话，决定打开率
- 要求：<140字符，有冲击力，制造好奇心或争议
- 技巧：数字开头 / 反直觉 / 提问 / 嘲讽

### 第四步：打分 & 排序

对每个角度评分（1-5 分）：

| 维度 | 权重 | 1分 | 5分 |
|------|------|-----|-----|
| 时效性 | 40% | 24-48h（已降级） | <6h 抢先窗口 |
| 数据支撑 | 30% | 纯观点无数据 | ≥3个可验证数字 |
| 争议性 | 20% | 无人在意 | 必引发讨论 |
| 话题热度 | 10% | 冷门 | 全网热议 |

**综合分 = 时效×0.4 + 数据×0.3 + 争议×0.2 + 热度×0.1**

⚠️ **注意**：>48h 的话题不进入打分，已在前置过滤步骤直接淘汰。

按综合分降序排列所有角度。

### 第五步：输出推荐

选出 **Top 3-5 个角度**（可能来自不同话题），输出给 tweet-composer。

## 与上下游的衔接

### ← 从 trend-scout 接收

topic-engine 重点关注 trend-scout 输出中的以下信号：

```
必看：
  - 🔥 必关注 Top 3-5（已排序的高价值热点）
  - 🐋 KOL vs 大户分歧矩阵（天然的推文素材）
  - 📊 数据信号（资金费率异常、量价异动、解锁事件）

参考：
  - 💬 社区风向（CT 情绪温度）
  - 🏛️ 宏观动态（今日是否有CPI/FOMC等催化剂）
  - ⚠️ 未验证信息（单源信息慎用，可作为"传闻"角度）
```

### → 输出给 tweet-composer

每个推荐角度包含 tweet-composer 需要的全部信息：
- 核心观点（写什么）
- 数据支撑（用什么数据）
- 开头金句（怎么开头）
- 发布形式建议（单推/Thread/引用推）
- 反向风险（必须包含的平衡视角）

tweet-composer 拿到后直接撰写，无需再做选题判断。

## 关键原则

- **时效硬截止**：>48h 直接淘汰，不开角度，不打分；24-48h 降级保留需有数据支撑
- **时效 > 一切**：6小时的话题比24小时的话题价值高10倍，速度优于完美
- **每个角度必须有反向风险**：体现思考深度，也是账号人设的核心特征
- **绝不预测价格**：可以说趋势、说风险、说逻辑，但不说"BTC会涨到X万"
- **数据为王**：有数字的角度永远排在纯叙事角度前面
- **分歧 = 价值**：KOL看空但大户做多，这种分歧本身就是最好的推文素材
- **不重复采集**：优先用 trend-scout 已有数据，只在必要时回调 MCP 补充
- **开头金句决定成败**：用户3秒内决定是否继续读，第一句必须有冲击力
