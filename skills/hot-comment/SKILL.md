# 热评制造机 Hot Comment Generator

CT曝光引擎。在高互动推文下抢高质量评���位，引流到自己账号。

核心目标：**速度快、有干货、活人感强、让人想点进你主页**。

---

## 核心定位

```
trend-scout（发现热点）→ 【hot-comment（发现目标+写评论）】→ engagement-manager（跟踪效果）
```

hot-comment 有两个职责：
1. **发现值得评论的高互动推文**（主动猎取目标）
2. **生成高质量评论**（抢占前排）

---

## 工作流程

### 第一步：发现目标推文

两种模式：

**A. 用户直接给推文** — 跳过发现环节，直接进入第二步

**B. 主动发现高互动推文** — 调用 MCP 工具：

```
策略1：KOL最新推文扫描
  工具：twitter_user_last_tweets
  参数：userName="[KOL用户名]", includeReplies=false
  目标：找到发布 <2h 且互动量上升中的推文
  
  优先扫描的KOL（参考 source-list.md）：
  - 大V类：CryptoHayes, DegenSpartan, WClementeIII, inversebrah
  - 中文类：Phyrex_Ni, 0xCryptowizard, CZ
  - 媒体类：WuBlockchain, TheBlock, CoinDesk
  → 选发布 <2h + 已有50+互动的推文作为目标

策略2：热点话题下的高互动推文
  工具：twitter_advanced_search
  参数：query="[当前热点关键词]", queryType="Top"
  ⚠️ 必须用 Agent 子进程（返回~99K字符）
  目标：找到话题下 views最高 但评论区还没饱和的推文

策略3：从 trend-scout 简报中提取
  trend-scout 的 Twitter 搜索结果已包含高互动推文
  直接复用，无需重新调用
```

**目标推文筛选标准：**

| 条件 | 说明 |
|------|------|
| 发布时间 <2h | 越新越好，>4h基本抢不到前排 |
| 已有互动 ≥50 likes | 说明推文在起势，值得评论 |
| 评论数 <100 | 评论区未饱和，你的评论还能被看到 |
| 作者粉丝 >10K | 曝光量足够大 |
| 话题与你账号相关 | 加密/宏观/交易相关，不蹭无关热点 |

### 第二步：分析目标推文

```
目标分析：
- 原推核心观点：[一句话]
- 原推作者定位：[KOL/项目方/交易员/媒体/...]
- 当前评论区氛围：[赞同为主/争议/嘲讽/求解释/...]
- 发布时间：[X分钟前] → 时间窗口判断
- 你的切入机会：[补充数据/反面观点/深挖细节/幽默点评/...]
```

**⏰ 时间窗口判断：**

| 发布时间 | 策略 | 评论数量 |
|---------|------|---------|
| <30min | 🔴 紧急抢位 — 速度第一，出1条最强评论 | 1条 |
| 30min-2h | 🟡 黄金窗口 — 质量+速度，出2-3条选最佳 | 2-3条 |
| 2h-4h | 🟢 正常模式 — 质量优先，多角度 | 3-5条 |
| >4h | ⚪ 低价值 — 除非评论区有新争议可加入 | 0-1条 |

### 第三步：生成评论

根据时间窗口决定数量，按类型确保多样性：

#### 📊 深度见解型（核心，占比最高）

展示专业能力，让人觉得"这人懂行"。

**要求：**
- 提供原推没提到的补充视角或数据
- 逻辑链：观察 → 推理 → 结论
- 长度：2-4句话

**写法参考：**
```
A. "补充一个数据：[具体数字]。从这个角度看，[你的推理]。关键变量是[X]。"
B. "同意大方向，但[具体环节]可能被低估了。[数据]。"
C. "这里有个细节：[细节]。历史上类似情况出现过[X次]，结果是[Y]。"
D. "[反向思考]。不是说观点错了，而是还有一个风险没定价：[风险]。"
```

**数据来源 — 按需回调 MCP：**
```
需要链上数据 → open_channel_feeds(quant_signal/fund_movement)
需要实时价格 → crypto_realtime_price_batch
需要KOL持仓 → kol_call_orders_24h / whale_trader_feeds
需要宏观数据 → finance_tool_treasury_rates
需要技术指标 → market_analyst(symbol, indicators="all")
```
优先用 trend-scout 已采集的数据，只在需要补充时回调。

#### 🔥 犀利观点型

强观点输出，符合账号人设。

**要求：**
- 一句话表达清晰立场
- 可以尖锐但不人身攻击
- 有逻辑支撑，不是纯情绪

**写法参考：**
```
A. "说白了就是[一句话犀利总结]。"
B. "[现象]说明什么？说明[犀利判断]。散户永远最后知道。"
```

#### 💬 互动引导型

设计成容易引发回复的评论，增加二次曝光。

**要求：**
- 以提问或半开放式判断结尾
- 让原作者或围观者有回复欲望
- 语气自然像在聊天

**写法参考：**
```
A. "这让我想到一个问题：[问题]？有人研究过吗？"
B. "如果[假设条件]，这个逻辑还成立吗？"
```

#### 😤 幽默/吐槽型

轻松有趣，增加"活人感"。

**要求：**
- 真的好笑或有洞察力，不是尬笑
- 可以自嘲、嘲讽现象，不嘲讽个人
- 简短：1-2句话

**写法参考：**
```
A. "[荒谬现象的幽默总结]。币圈永远不让人失望。"
B. "看完这条我的反应：[简短反应]。然后看了眼持仓，又冷静了。"
```

**评论数量分配（按时间窗口）：**

| 时间窗口 | 总数 | 深度 | 犀利 | 互动 | 幽默 |
|---------|------|------|------|------|------|
| 🔴 <30min | 1条 | 1 | - | - | - |
| 🟡 30min-2h | 3条 | 1 | 1 | 1 | - |
| 🟢 2h-4h | 5条 | 2 | 1 | 1 | 1 |

紧急模式只出1条深度见解 — 因为深度型最容易获得原作者回复和关注。

### 第四步：去AI味检查

**硬性检查（每条必过）：**

- [ ] 读起来像真人在认真回复，不像模板
- [ ] 没有"首先/其次/最后"这种结构化表达
- [ ] 没有"值得关注"、"不容忽视"、"总的来说"等AI高频词
- [ ] 长度自然（不是每条都差不多长）
- [ ] 和原推内容直接相关，不是泛泛而谈

**去AI味替换表：**

| ❌ AI味 | ✅ 真人味 |
|--------|---------|
| 值得关注的是 | 有意思的是 / 关键在于 |
| 不容忽视 | 别忽略 / 这个被低估了 |
| 总的来说 | 说白了 / 一句话 |
| 首先...其次...最后 | 直接说要点，不编号 |
| 这是一个很好的观点 | （删掉，直接补充你的观点） |
| 我认为这非常重要 | 这个关键 / 这才是重点 |

## 与上下游的衔接

### ← 从 trend-scout 接收
- Twitter 搜索结果中的高互动推文（可直接作为评论目标）
- 热点话题关键词（用于 twitter_advanced_search 发现目标）
- 已采集的数据（价格/链上/KOL持仓，可直接用于数据型评论）

### → 输出给 engagement-manager
- 已发布的评论列表（追踪哪条获得了回复/点赞）
- 哪些 KOL 回复了你（高价值互动关系）

## 关键原则

- **速度 > 完美**：<30min的推文出1条就发，别等写完5条再选
- **活人感 > 完美措辞**：宁可口语化，不要像AI写的
- **有料 > 有趣**：深度见解是核心，幽默是加分项
- **简短有力**：评论区没人看长篇大论，2-4句话最佳
- **不舔不怼**：专业补充比"大佬说得对"有用100倍
- **针对性强**：每条评论必须和原推内容直接相关
- **数据加持**：有具体数字的评论 >> 纯观点评论
