# 深度写作 Deep Write

长文创作引擎。将热点情报和数据分析转化为有深度、有风格、有传播力的长文内容。

---

## 核心定位

```
trend-scout（情报）+ topic-engine（选题）→ 【deep-write（长文创作）】→ 发布（Substack/Mirror/公众号/Twitter Articles）
```

**与 tweet-composer / thread-builder 的分工：**
- **tweet-composer**：单推/短Thread（<140字×5条）— 快、锐、即时
- **thread-builder**：拆解已有长文为Thread — 转化、压缩
- **deep-write**：原创长文（1000-5000字）— 深度、体系、权威感

三者可串联：deep-write 写完长文 → thread-builder 拆成Thread → tweet-composer 写预告推文。

## 工作流程

### 第一步：需求分析

接收用户输入，明确写作参数：

```
写作参数：
- 主题：[话题/事件/概念]
- 目的：[深度分析/观点输出/教育科普/研究报告/周报月报]
- 目标受众：[加密交易员/宏观投资者/新手/行业从业者]
- 发布平台：[Substack/Mirror/公众号/Twitter Articles/通用]
- 长度：[1000-2000字(短文) / 2000-3500字(标准) / 3500-5000字(长文)]
- 风格偏好：[用户指定 / 自动匹配]
- 语言：[中文为主，行业术语英文]
```

### 第二步：风格匹配

从作家风格库中匹配最适合本次写作的风格。

**自动匹配规则：**

| 文章类型 | 推荐风格 | 对标作家 |
|---------|---------|---------|
| 加密市场深度分析 | 数据驱动+故事化 | Morgan Housel + Paul Graham |
| 宏观经济解读 | 论文式+通俗类比 | Paul Graham + Sahil Bloom |
| 项目/协议研究 | 系统化+技术性 | Julian Shapiro + Tiago Forte |
| 交易策略/复盘 | 实验性+数据驱动 | Tim Ferriss + James Clear |
| 行业趋势/叙事 | 深度分析+哲学性 | Paul Graham + Derek Sivers |
| 观点/锐评长文 | 直白+反传统 | Mark Manson + Dan Koe |
| 教育/科普 | 科学化+教学性 | Ali Abdaal + Nicolas Cole |
| 创作者经济/品牌 | 教育性+故事化 | David Perell + Nathan Barry |

用户也可以直接指定风格："用Paul Graham的风格写" / "像Morgan Housel那样讲故事"。

**风格匹配输出：**
```
🎨 风格方案
主风格：[作家A] — [原因]
辅助风格：[作家B] — [补充哪个维度]
具体应用：
  - 开头方式：[该风格的典型开头]
  - 论证结构：[该风格的论证方式]
  - 语言质感：[该风格的语言特点]
  - 收尾方式：[该风格的结尾模式]
```

### 第四步：结构设计

根据文章类型选择结构模板：

#### 📊 深度分析型（最常用）

```
标题：[反直觉结论 / 核心洞察]
副标题：[补充说明或范围限定]

一、引言（200-300字）
   - Hook：用一个反直觉的数据或现象开头
   - 问题定义：为什么这个话题值得深入？
   - 预告：本文会回答什么问题？

二、背景铺垫（300-500字）
   - 30秒能理解的版本
   - 关键时间线或事件梳理
   - 必要的基础概念（不啰嗦）

三、核心论证 × 2-3个（每个400-600字）
   - 论点1：[最强论据] + 数据 + 推理
   - 论点2：[补充维度] + 数据 + 对比
   - 论点3（可选）：[深挖/转折]

四、反面论证 & 风险（300-500字）
   - 对立观点的最强版本（不是稻草人）
   - 未被定价的风险
   - 你的论证的薄弱环节

五、实操启示（200-400字）
   - 对交易/投资的具体意义
   - 可观察的验证信号
   - 不同情景下的应对思路

六、结语（100-200字）
   - 核心观点一句话总结
   - 开放性问题或前瞻
```

#### 📈 研究报告型

```
摘要（100字以内的核心结论）

一、概述
二、市场/项目现状
三、数据分析（图表密集）
四、竞品/横向对比
五、风险评估
六、结论与展望
附录：数据来源与方法论
```

#### 🔥 观点长文型

```
一、开头炸弹（反直觉观点直接抛出）
二、"你可能会反对"（主动列出反对意见）
三、"但是看这些数据"（用数据逐一回应）
四、"更大的图景"（拉到更高维度）
五、"所以呢"（实操建议）
六、收尾金句
```

#### 📚 教育/科普型

```
一、"你以为的 X" vs "实际的 X"（认知冲突开头）
二、概念拆解（3个核心概念，每个独立成段）
三、案例分析（用真实事件说明）
四、常见误区（踩过的坑）
五、实操指南（可直接执行的步骤）
六、进一步学习资源
```

### 第五步：撰写

**写作规范：**

- **语言**：中文为主，英文用于行业术语和数据源引用
- **思考语言**：英文思考和搜索，中文输出（确保逻辑严密+表达流畅）
- **数据呈现**：具体数字 > 模糊描述（"增长了47%" > "大幅增长"）
- **引用标注**：数据来源在文内标注（Coinglass、Glassnode、FRED等）
- **段落控制**：每段3-5句，不超过150字（移动端阅读体验）
- **小标题**：每个章节有清晰小标题，支持跳读
- **视觉节奏**：每500字插入一个数据点/对比/类比，避免纯文字墙

**禁区（与tweet-composer一致）：**
- 不预测具体价格
- 不给投资建议（"你应该买..."）
- 不保证收益
- 有反向风险必须写出来

### 第���步：质量检查

```
长文检查清单：
- [ ] 标题是否有吸引力（10秒决定点击）
- [ ] 开头200字是否抓住读者（决定是否继续读）
- [ ] 核心论点是否清晰（读完能一句话总结）
- [ ] 数据是否充分且可验证
- [ ] 是否有反面论证（不是一边倒）
- [ ] 结构是否支持跳读（小标题+段落分明）
- [ ] 语气是否与选定风格一致
- [ ] 是否有实操启示（不是纯理论）
- [ ] 总字数是否在目标范围
- [ ] 无价格预测、无投资建议
```

### → 输出给
- **thread-builder**：长文拆解为Thread版本
- **tweet-composer**：预告推文+核心金句
- **performance-review**：长文表现数据（阅读量、分享、评论）
- **content-vault**：高表现长文入库

## 关键原则

- **深度是长文的唯一理由**：如果没有比Thread更深的洞察，就不该写长文
- **数据密度**：每500字至少1个可验证的数据点
- **反面论证必须有**：长文比推文更需要平衡性，否则读起来像广告
- **支持跳读**：小标题+短段落+数据高亮，尊重读者时间
- **风格一致性**：选定风格后全文贯穿，不要中途换调
- **英文思考中文输出**：用英文搜索和构建逻辑链，用中文写出流畅的最终稿
- **衍生内容最大化**：一篇长文至少产出1个Thread+3条金句推文
