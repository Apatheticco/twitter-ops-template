# 内容素材库 Content Vault

你的推文弹药库。从历史推文数据中挖掘高表现内容，提炼可复用的模式，建立一个持续增长的素材库。写新推文时不用从零开始，直接从弹药库里调配。

## 两种使用模式

### 模式 A：首次建库（扫描历史推文）

第一次使用时，需要扫描你的历史推文，建立基础素材库。

**第一步：获取历史推文数据**

通过 MCP 工具拉取推文：

```
主要工具：
  twitter_user_last_tweets
  参数：userName="[你的用户名]"
  拿到：近期推文列表 + 互动数据（likes/retweets/replies/views）

补充工具（按需）：
  twitter_tweets_by_ids — 获取指定推文的最新数据
  twitter_user_timeline — 获取更长时间范围的推文

需要的数据字段：
- 推文内容（text）
- 发布时间（created_at）
- 点赞数（likeCount）
- 转发数（retweetCount）
- 评论数（replyCount）
- 引用数（quoteCount）
- 浏览数（viewCount）
- 是否为线程（conversationId 判断）
```

如果 MCP 工具拉取不足，用户也可以：
- 提供 Twitter 数据导出文件（Settings → Your Account → Download Archive）
- 手动提供推文数据
- 直接告诉我哪些推文表现好

**第二步：筛选高表现推文**

用互动率（非绝对数字）筛选，因为粉丝量在涨，早期推文绝对数低但比率可能高：

```
互动率 = (点赞 + 转发 + 评论 + 引用) / 印象数

筛选标准：
- 🏆 S级（互动率 Top 10%）→ 必须入库，深度拆解
- ⭐ A级（互动率 Top 25%）→ 入库，提炼模式
- 📌 值得关注（互动率高于平均 1.5 倍）→ 标记观察
```

**第三步：拆解每条高表现推文**

对筛出的每条推文做结构化分析：

```
### 🏆 [推文摘要前15字...]

**原文**：[完整推文内容]
**发布时间**：YYYY-MM-DD HH:MM
**数据**：❤️ X | 🔄 X | 💬 X | 👁️ X | 互动率 X%

**拆解**：
- **话题类别**：[BTC行情/链上数据/宏观分析/项目点评/行业吐槽/...]
- **角度类型**：[锐评/数据驱动/反向思考/嘲讽/教育/快讯反应]
- **开头技巧**：[反问/数据冲击/对立观点/自我否定/类比/...]
- **结构模式**：[单句/观点+数据+结论/问题+分析+反问/...]
- **钩子类型**：[提问/反转/CTA/悬念/...]
- **情绪基调**：[愤怒/兴奋/冷静分析/幽默/警告/...]
- **成功假设**：[为什么这条表现好？1-2句分析]

**可复用元素**：
- 句式模板：[提炼出可填空的句式]
- 数据框架：[如果用了数据，什么样的数据呈现方式有效]
- 话题关键词：[哪些关键词/话题自带流量]
```

**第四步：生成素材库**

将拆解结果整理成结构化素材库，存入 `references/vault.md`：

```
# 📦 我的推文素材库

更新时间：[日期]
总入库：X 条 | S级：X 条 | A级：X 条

## 按话题分类索引
### BTC/比特币 (X条)
- [摘要] — 角度：锐评 — 互动率：X%
- [摘要] — 角度：数据 — 互动率：X%

### 宏观经济 (X条)
- ...

### 链上数据 (X条)
- ...

## 按角度分类索引
### 锐评类 (X条)
### 数据驱动类 (X条)
### 反向思考类 (X条)
### 嘲讽/吐槽类 (X条)

## 高频成功模式 Top 5
1. [模式描述] — 出现 X 次 — 平均互动率 X%
2. ...

## 验证有效的句式模板
1. "所有人都在[共识]，但[反向数据/逻辑]" — 适用于：反向观点
2. "[数字] + [趋势] + [所以呢？]" — 适用于：数据帖
3. ...

## 高流量话题关键词
[按互动率排序的话题关键词列表]
```

### 模式 B：日常使用（调用素材库）

素材库建好后，在日常运营中调用：

**场景 1：写推文前找灵感**
```
输入："今天想写关于 ETH 的推文，有什么参考？"

→ 从素材库中搜索：
  - ETH 相关的历史高表现推文
  - 适用的角度和句式模板
  - 之前有效的钩子类型
→ 输出：3 个灵感方向 + 对应的历史参考
```

**场景 2：验证角度是否靠谱**
```
输入："我想用反向观点的角度写 BTC ETF"

→ 从素材库中查找：
  - 之前反向观点类推文的表现数据
  - 哪种反向手法最有效
  - 要避免的坑（如果之前某个反向角度翻车了）
→ 输出：历史数据支撑 + 建议调整
```

**场景 3：复用有效句式**
```
输入："给我几个开头模板"

→ 从素材库中提取：
  - 互动率最高的 5 个开头句式
  - 每个句式的适用场景和历史数据
→ 输出：可直接填空的模板列表
```

**场景 4：定期更新素材库**
```
输入："更新素材库" / "把上周的推文入库"

→ 拉取最近一周的推文数据
→ 筛选符合入库标准的推文
→ 拆解并追加到 vault.md
→ 更新"高频成功模式"和"句式模板"
```

## 素材库维护规则

- **入库标准**：互动率 Top 25%，或有明显可复用的模式
- **淘汰机制**：超过 6 个月没有被参考的素材标记为"冷库"
- **模式更新**：每月重新计算"高频成功模式"排名
- **反面教材**：互动率极低的推文也值得记录（标记为"避坑"），避免重蹈覆辙

## 与上下游的衔接

### ← 接收（入库来源）
- **performance-review**：周度/月度复盘中识别的高表现推文 → 自动入库
- **engagement-manager**：获得高互动的 Outbound 评论 → 入库（评论也是素材）
- **competitor-watch**：竞对爆款的角度分析 → 入库作为参考（标记为"竞对参考"）

### → 输出给（调用素材）
- **topic-engine**：历史话题表现数据 → 指导选题优先级
- **tweet-composer**：有效句式模板+高流量关键词 → 直接复用
- **hot-comment**：历史高表现评论的模式 → 指导评论写法
